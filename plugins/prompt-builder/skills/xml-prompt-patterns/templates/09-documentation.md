# Technical Documentation Template

```xml
Generate documentation for the following code.

<code language="{{LANGUAGE}}">
{{CODE_TO_DOCUMENT}}
</code>

<documentation-style>
{{STYLE_GUIDE}} (JSDoc, Google, NumPy, Sphinx, etc.)
</documentation-style>

<audience>
{{WHO_WILL_READ}} (developers, API users, team members)
</audience>

<instructions>
1. Add comprehensive docstrings/comments
2. Document parameters, returns, exceptions
3. Include usage examples
4. Note important caveats or edge cases
5. Follow the specified style guide
</instructions>

<output-format>
<documented-code>
[Code with inline documentation]
</documented-code>

<api-reference>
[Standalone API documentation]
</api-reference>

<examples>
[Usage examples with explanations]
</examples>
</output-format>
```

## Usage
Specify your documentation style for consistent formatting.
Indicate the target audience for appropriate detail level.
